## 🤔 Зачем это нужно?

Развернуть Apache вручную — значит часами разбираться в конфигах, устанавливать модули, настраивать VirtualHost, возиться с certbot и UFW. Этот скрипт делает всё сам:

- 🌐 **VirtualHost** — создаётся автоматически с нужным доменом и путями
- 🔒 **SSL-сертификат** — получается через Let's Encrypt, обновляется по cron
- 🛡️ **Безопасность** — security-заголовки, скрытие версии, блокировка служебных файлов
- 🗄️ **База данных** — MariaDB, MySQL или PostgreSQL из официального репо
- ⚡ **Производительность** — Gzip, кеш браузера, HTTP/2 из коробки

## ✨ Возможности

### 🐘 PHP

Опциональная установка с выбором версии и набора расширений. Версии не из системного репо подключаются через PPA `ondrej/php` автоматически.

**Доступные версии:**

| Ubuntu | Версии PHP |
|--------|-----------|
| 20.04 Focal | 7.4, 8.0, 8.1, 8.2, 8.3, **8.4** |
| 22.04 Jammy | 7.4, 8.0, 8.1, 8.2, 8.3, **8.4** |
| 24.04 Noble | 8.1, 8.2, 8.3, **8.4** |

**Наборы расширений:**

- **Минимальный** — cli, mysql, curl, mbstring, xml, zip
- **Стандартный** — + gd, intl, bcmath, opcache, redis
- **Максимальный** — + imagick, soap, ldap, xsl, dev

### 🗄️ Базы данных

| СУБД | Версии | Источник |
|------|--------|----------|
| **MariaDB** | 10.6 LTS · 10.11 LTS · 11.4 LTS | mariadb.org |
| **MySQL** | 8.0 · 8.4 LTS | repo.mysql.com |
| **PostgreSQL** | 14 · 15 · 16 · 17 | apt.postgresql.org |

Для каждой СУБД автоматически:

### 🔒 SSL / HTTPS

Сертификаты **Let's Encrypt** через **Certbot** — устанавливается только через snap (официальный рекомендуемый способ EFF). Если snapd отсутствует — устанавливается автоматически.

- ✅ Автоматическое получение сертификата
- ✅ Редирект HTTP → HTTPS
- ✅ HSTS (`Strict-Transport-Security: max-age=31536000; includeSubDomains; preload`)
- ✅ Cron-автообновление 1-го и 26-го числа каждого месяца в 03:00

### ⚡ Производительность

- **Gzip** — `mod_deflate` для HTML, CSS, JS, JSON, SVG
- **Кеш браузера** — `mod_expires`: изображения 1 год, CSS/JS 1 месяц
- **HTTP/2** — `mod_http2`
- **MPM** — `event` (рекомендуется), `prefork` (для mod_php), `worker`

---

### 📄 Контент и логи

- `index.html` — стильная тёмная заглушка с анимированным индикатором статуса
- `robots.txt` — открытый или закрытый (режим «сайт в разработке»)
- `.htaccess` — защита файлов, rewrite-правила, сжатие, кеш
- Страницы ошибок **403 / 404 / 500** в едином тёмном дизайне
- Ротация логов через `logrotate` с настраиваемым сроком хранения

---

### 🔥 Firewall (UFW)

- Автоустановка UFW если отсутствует
- Открытие портов 80 + 443 (`Apache Full`)
- Опциональное разрешение SSH (порт 22)


## 📁 Что создаётся в итоге

```
/etc/apache2/sites-available/<домен>.conf    # VirtualHost
/var/www/<домен>/public_html/
├── index.html                               # Заглушка (опционально)
├── robots.txt                               # (опционально)
├── .htaccess                                # (опционально)
└── errors/
    ├── 403.html
    ├── 404.html
    └── 500.html

/root/.db_<домен>                            # Данные БД (chmod 600)
/var/log/apache-setup-<дата>.log             # Лог установки
/var/log/apache2/<домен>_access.log
/var/log/apache2/<домен>_error.log
/etc/logrotate.d/apache2-<домен>             # Ротация логов
```

## 🔐 Безопасность реализации

Скрипт написан с акцентом на надёжность:

- **Проверка dpkg lock** — ждёт освобождения до 120 секунд перед установкой пакетов
- **Блокировка `apt-daily`** — таймеры останавливаются на время установки и восстанавливаются через `trap EXIT`
- **Snap-only certbot** — apt-версия certbot при наличии удаляется во избежание конфликтов путей
- **Безопасная генерация пароля** — через `openssl rand` с резервным fallback на `/dev/urandom`, с проверкой что пароль непустой
- **Все действия логируются** — в `/var/log/apache-setup-<дата>.log`
